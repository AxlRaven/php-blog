# PHP Blog

Лёгкий движок персонального блога на PHP + MySQL без фреймворков.  
Markdown-записи, админка, SEO, 2FA, загрузка медиа — всё в одном дистрибутиве.

**Версия схемы БД:** 36  
**Лицензия кода сторонних библиотек:** см. файлы в `vendor/` (Parsedown, php-qrcode).

---

## Возможности

### Публичная часть
- Лента записей с **пагинацией** или кнопкой **«Показать ещё»**
- Разрыв «Читать далее»: `[more]` / `<!--more-->`
- Страница записи, **теги**, **поиск** с подсветкой найденных слов
- Счётчик **просмотров** и **рейтинг** 1–5 ★ (с `aggregateRating` в SEO)
- Обрезка высоких картинок в ленте, центрирование узких, **лайтбокс**
- Адаптивная вёрстка

### Редактор и медиа
- Markdown-редактор **EasyMDE**
- Загрузка изображений и файлов, вставка из буфера (**Ctrl+V**), drag & drop
- Миниатюры через **GD**
- Подсветка кода (**Prism**: PHP, JS, CSS, SQL, Python, Bash, JSON, Markup)

### SEO и интеграции
- Open Graph / Twitter Card, JSON-LD
- `sitemap.xml`, `robots.txt`, RSS (`/feed`)
- Канонический URL, ключевые слова, автор, организация
- Коды верификации Google Search Console и Яндекс.Вебмастер
- Логотип и обложка (cover) для соцсетей

### Админка и безопасность
- Вход администратора, CSRF-защита
- **TOTP 2FA** (Google Authenticator и аналоги), QR в интерфейсе
- Загрузка **logo.png** и **cover.jpg** из настроек
- Предупреждение, если на сервере остались `install.php` или папки `update_*`
- Защита каталогов `config/`, `includes/`, `vendor/`; запрет PHP в `uploads/`

## Требования

| Компонент | Минимум |
|-----------|---------|
| PHP | 8.0+ (рекомендуется 8.1+) |
| БД | MySQL 5.7+ / MariaDB 10.3+ |
| Веб-сервер | Apache с `mod_rewrite`, `AllowOverride All` |
| PHP-расширения | `pdo_mysql`, `fileinfo`, `mbstring`, **gd** |

Права на запись нужны для:
- `config/` (создание `config.php` при установке)
- `uploads/` и подкаталогов (`images`, `images/thumbs`, `files`)

## Установка

1. Скачайте репозиторий или архив и залейте **содержимое** в корень сайта (или в подкаталог).
2. Убедитесь, что `uploads/` доступна для записи веб-серверу (`chmod 775` или владелец = пользователь PHP-FPM/Apache).
3. Откройте в браузере:  
   `https://ваш-домен/install.php`
4. Заполните форму:
   - хост, имя БД, пользователь и пароль MySQL;
   - название сайта;
   - логин и пароль администратора (не короче 6 символов).
5. После успешной установки **обязательно удалите `install.php`** с сервера.
6. Войдите: `https://ваш-домен/admin/login`

Установщик сам создаёт таблицы, настройки и проверяет возможность записи в `uploads/`.

### Пример прав (Linux)

```bash
chown -R www-data:www-data uploads config
chmod 775 uploads uploads/images uploads/images/thumbs uploads/files
chmod 775 config
```

После установки `config/config.php` лучше сделать недоступным для записи всеми, кроме владельца (`chmod 640`).

## Структура проекта

```
├── admin/           # Админ-панель
├── api/             # RSS, sitemap, robots, рейтинг, лента JSON
├── assets/          # CSS, JS, шрифты, изображения
├── config/          # config.sample.php → config.php при установке

├── uploads/         # Загрузки (не коммитить пользовательский контент)
├── vendor/          # Parsedown, генератор QR
├── index.php        # Главная
├── post.php         # Запись
├── tag.php / search.php
├── install.php      # Одноразовая установка (удалить после)
└── .htaccess        # ЧПУ и защита каталогов
```

## Настройки (Админка → Настройки)

### Сайт
- **Название** и **описание** (для SEO и соцсетей)

### Логотип и обложка
| Файл | Куда сохраняется | Зачем |
|------|------------------|--------|
| Логотип | `uploads/images/logo.png` | Шапка + favicon (лучше квадрат 256×256) |
| Cover | `uploads/images/cover.jpg` | Картинка по умолчанию для OG / Telegram |

Форматы: JPEG, PNG, GIF, WebP (до 5 МБ). Нужен GD.  
Альтернатива: положить `assets/images/logo.png` вручную — используется, если в `uploads` логотипа нет.

### SEO
- Канонический **URL сайта** (`https://…`)
- Ключевые слова, автор, организация, Twitter/X
- Путь к OG-картинке (при загрузке cover подставляется сам, если поле пустое)
- Коды Google / Яндекс

Полезные URL после установки:
- RSS: `/feed`
- Sitemap: `/sitemap.xml`
- Robots: `/robots.txt`

Telegram кэширует превью — обновите карточку через [@WebpageBot](https://t.me/WebpageBot).

### Лента
- Число записей на странице
- Режим: **постранично** или **«Показать ещё»**

### Двухфакторная аутентификация (TOTP)
1. **Настроить 2FA** → отсканируйте QR или введите ключ вручную в приложении.
2. Введите 6-значный код → **Подтвердить и включить**.
3. При входе: пароль, затем код из приложения.
4. Отключение: пароль + текущий TOTP-код.

### Смена пароля
Поля «Новый пароль» / «Повтор» внизу формы настроек (оставьте пустыми, если не меняете).

Нажмите **Сохранить**, чтобы записать изменения сайта, SEO, ленты, брендинга и пароля.

## Работа с записями

### Создание и редактирование
1. **Записи** → **Новая запись** (или редактирование существующей).
2. Заголовок, slug (можно оставить авто), текст в Markdown.
3. Теги через запятую.
4. Статус: **черновик** или **опубликовано**; можно задать дату публикации (отложенный выход).
5. Сохраните.

### Markdown и вставки
- Заголовки, списки, ссылки, код — обычный Markdown
- Разрыв превью: строка `[more]` или `<!--more-->`
- Картинки/файлы — кнопки редактора или вставка из буфера
- Блоки кода с языком для Prism, например:

````markdown
```php
echo 'Hello';
```
````

### Черновики и превью
Неопубликованные записи видны в админке; для проверки используйте ссылку превью из интерфейса редактирования (если доступна в вашей сборке).

## Безопасность после установки

1. Удалите `install.php`.
2. Не оставляйте на сервере папки/скрипты `update_*`.
3. Включите **2FA** для администратора.
4. Не публикуйте `config/config.php` (пароль БД).
5. Каталог `uploads/` должен быть writable, но выполнение PHP в нём запрещено (есть `.htaccess`).

В админке при наличии `install.php` или `update_*` показывается предупреждение.

## Конфигурация вручную

Образец: `config/config.sample.php`.  
Рабочий файл `config/config.php` создаётся установщиком и обычно выглядит так:

```php
<?php
return [
    'db' => [
        'host' => 'localhost',
        'name' => 'blog_db',
        'user' => 'user',
        'pass' => 'secret',
        'charset' => 'utf8mb4',
    ],
    'site' => [
        'title' => 'Мой блог',
        'description' => '',
        'url' => '',
    ],
    'installed' => true,
];
```

## Разработка и деплой

- Дистрибутив самодостаточен: отдельные `update_*.php` для **новой** установки не нужны.
- Для уже работающего сайта обновления накатываются пакетами по порядку (вне этого репозитория, если вы ведёте их отдельно).
- Не коммитьте `config/config.php` и содержимое `uploads/` (кроме `.gitkeep` / `.htaccess`).

## Краткий чеклист первого запуска

1. Залить файлы → права на `uploads` и `config`
2. Открыть `/install.php` → заполнить форму
3. Удалить `install.php`
4. Войти в `/admin/login`
5. Настройки: URL сайта, логотип, cover, SEO
6. Включить 2FA
7. Создать первую запись

## Поддержка окружения

Если картинки не загружаются или нет миниатюр — проверьте расширение **GD** и права на `uploads/images`.  
Если ЧПУ не работают — включите `mod_rewrite` и `AllowOverride All` для каталога сайта.
